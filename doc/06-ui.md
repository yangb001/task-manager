# 七、UI 界面设计（v2 — 现代化风格）

## 7.1 设计原则

- **毛玻璃质感**（Glassmorphism）：卡片半透明模糊背景，层次分明
- **暗色/亮色双主题**：支持一键切换，背景色/背景图可自定义
- **响应式布局**：左侧导航 + 右侧内容区，任务卡片网格自适应
- **微交互动效**：hover 缩放、状态切换过渡、加载骨架屏
- **信息密度适中**：关键信息一目了然，详情可展开

## 7.2 整体布局

左侧导航（200px 毛玻璃）+ 顶部栏（标题/搜索/主题切换）+ 内容区（状态 Tab + 卡片网格）+ 底部状态栏

### 布局说明

| 区域 | 说明 |
|---|---|
| **左侧导航** | 固定宽度 200px，半透明毛玻璃背景，图标+文字菜单 |
| **顶部栏** | 标题 + 搜索框 + 主题切换(🌓) + 设置入口(⚙) |
| **内容区** | 状态 Tab + 任务卡片网格（2~3 列自适应） |
| **底部状态栏** | 引擎状态、运行中数量、触发器活跃数 |

## 7.3 主题与个性化

### 主题系统

```typescript
interface ThemeConfig {
  mode: 'light' | 'dark';
  primaryColor: string;
  backgroundType: 'color' | 'image' | 'gradient';
  backgroundColor: string;
  backgroundImage: string;
  backgroundGradient: string;
  backgroundBlur: number;       // 0-20px
  cardOpacity: number;          // 0.6-1.0
  borderRadius: number;         // 4-20px
}
```

### 主题设置

模式切换（亮色/暗色）、主色调选择器（预设色 + 自定义）、背景类型（纯色/渐变/图片，图片支持透明度调节）、卡片样式（不透明度/圆角/模糊度）、预设主题一键切换（默认亮色/深邃暗色/极简白）

## 7.4 任务卡片设计（毛玻璃质感）

每张卡片显示：状态图标 + 任务名 + 类型/调度信息 + 触发→动作摘要 + 上次执行状态 + 操作按钮

**卡片交互：**
- 默认：半透明白色/暗色背景，轻微阴影
- Hover：轻微上浮（translateY -2px），阴影加深
- 运行中：左侧绿色脉冲边框动画
- 失败：左侧红色边框 + 错误信息高亮
- 暂停：灰色调，操作按钮显示"▶ 恢复"

## 7.5 新建/编辑任务弹窗（分步式）

### 步骤 1/3 — 基本信息

步骤指示器（● 基本信息 ──── ○ 调度与触发 ──── ○ 动作与输出）

表单：任务名称、描述（多行）、标签（标签选择器，可多选+自定义）、分组（下拉选择或新建）

底部实时预览卡片效果。

### 步骤 2/3 — 调度与触发

调度设置：单次任务（立即执行 / 指定时间）和定时任务（Cron 表达式 + 开始/结束时间 + 最大次数）并排展示。

触发器区域：
- 关系选择（OR/AND）
- 添加触发器下拉菜单（从 `plugins.listTriggers()` 动态获取可用插件列表，显示插件名称+图标）
- 选择插件后，使用 `<TriggerConfig>` 组件渲染动态配置表单（基于插件 manifest 中的 `config_schema` JSON Schema 自动生成）
- 顶部显示插件图标、名称、描述
- 配置表单支持的字段类型：string（text/textarea）、number、boolean、enum（select）、object（嵌套分组）
- 每个字段显示 title（label）、description（help text）、required 标记、default 值占位
- 保留删除按钮

### 步骤 3/3 — 动作与输出

动作序列：
- 添加动作下拉菜单（从 `plugins.listActions()` 动态获取可用插件列表，显示插件名称+图标）
- 选择插件后，使用 `<ActionConfig>` 组件渲染动态配置表单（同触发器，基于 JSON Schema 自动生成）
- 额外字段区域（执行选项）：
  - 输出变量名（`output_var`）：后续可通过 `{{steps.xxx}}` 引用
  - 失败时继续执行（`continue_on_error`）：checkbox 开关
- 拖拽排序手柄（后续迭代）

高级设置折叠面板：失败重试、超时时间、并发限制。

### JSON Schema 表单渲染器

`<SchemaForm>` 组件根据 JSON Schema 自动渲染表单：

| Schema 类型 | 渲染组件 |
|---|---|
| string | `<input type="text">` |
| string + format:textarea | `<textarea>` |
| string + enum | `<select>` |
| number / integer | `<input type="number">`（支持 minimum/maximum） |
| boolean | `<input type="checkbox">` |
| object（有 properties） | 内联展示子字段（递归渲染） |

## 7.6 插件管理页面

独立的插件管理页面，集中查看和管理所有触发器和动作插件。

### 页面结构

```
┌─────────────────────────────────────────────────────────┐
│  插件管理                                    [扫描插件]   │
├─────────────────────────────────────────────────────────┤
│  [触发器 (N)]  [动作 (N)]                                │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐│
│  │ 🌐 HTTP 远程调用      v1.0  [系统内置]   使用中: 3   ││
│  │ 发送 HTTP/HTTPS 请求到远程 API                       ││
│  ├─────────────────────────────────────────────────────┤│
│  │ 💬 WeLink 消息        v1.0  [系统内置]   使用中: 1   ││
│  │ 发送文本或 Markdown 消息到 WeLink                    ││
│  ├─────────────────────────────────────────────────────┤│
│  │ 🌐 Webhook            v1.0  [系统内置]   未使用      ││
│  │ 通过 HTTP 请求触发任务                               ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 功能说明

- **Tab 切换**：触发器 / 动作，显示各类型数量
- **插件卡片**：显示图标、名称、版本、来源标签（系统内置 / 用户插件）、描述、分类、使用中任务数
- **展开详情**：点击卡片展开
  - 描述信息
  - 权限标签（network / filesystem / process / vault / env）
  - 配置 Schema 预览（只读 JSON）
  - 关联任务列表（点击可跳转到任务列表页）
  - 用户插件显示"卸载"按钮
- **扫描插件**：重新扫描用户插件目录，加载新安装的插件

## 7.7 执行历史页面

状态筛选 Tab（全部/成功/失败/运行中）+ 按任务筛选下拉。列表展示每次执行：任务名、时间、状态、耗时、动作链摘要、错误信息（如有）。点击 [详情] 打开执行详情弹窗。

## 7.8 执行详情弹窗

三个区域：
1. **基本信息**：状态、耗时、起止时间、触发方式、触发数据
2. **执行步骤**：每个动作一个卡片，显示配置、输出、错误、耗时。可展开查看详细日志时间线（时间戳 | 级别 | 消息）
3. **变量作用域**：可展开查看 JSON 快照

底部操作：[重试此任务] [导出日志] [删除此记录]

## 7.9 系统日志页面

筛选条件：级别、模块、时间范围、关键词搜索。列表展示每条日志：时间、级别（颜色标签）、模块、消息。点击展开详情。

## 7.10 设置页面

四个 Tab：账号管理、主题设置、日志设置、关于

- **账号管理**：WeLink 账号列表（名称/Token 状态/操作）、邮箱账号列表（IMAP/SMTP 配置/连接状态）、LLM Provider 列表（模型/API Key 状态）
- **主题设置**：见 7.3 节
- **日志设置**：保留天数/条数、文件日志开关、Debug 日志开关、立即清理按钮、当前占用空间

## 7.11 系统托盘

右键菜单：打开主窗口、运行中任务列表（点击跳转）、最近执行结果、退出

关闭窗口 → 隐藏到托盘，主进程继续运行。执行失败时托盘图标闪烁 + 系统通知。
